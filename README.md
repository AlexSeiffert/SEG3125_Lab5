# SEG3125_Lab5

This is for Lab 5. It is a bike repair shop called "Alex Cycle Shop".  It was created with Bootstrap 5.

## Description from Lab 4
The site allows users to book different services and see the prices of those services.  The workflow is clear to the user.  The user can start the booking by selecting a service, then picking a date and time and including their contact info.  After that, they are able to confirm their booking and a modal pops up to confirm their booking.

## New Requirements

- The user can choose the professional of their choice in Step 1, when choosing the mechanic.
- A payment section has been added to where the user can provide their credit card information
- There is a nav bar that jumps to different sections on the same page of the website (the website only has one page)
- Similar elements were also used on the website.
- Icons were added to Services (Icon to show the estimated amount of time), Experts (Icon to indicate any 'selling points' about the specific mechanic such as 15+ years of experience), and Payments (Icons that go along with the fields, such as a card icon to go with the "Card number" field).
- An icon was added to the error message if a user supplied the form with an invalid input
- Icons were added in Step 3 when choosing the date & time (e.g. a calendar for the "Date (Mon-Sat)" field)
- Icons were also added when displaying the store's Address, Hours, and Contact information.
- Tooltips were used for the user to understand what information to provide, why it may be needed, and/or additional information for the user.
- Input fields may be highlighted when clicked on
- Regular expresisons are used to regulated input fields for the Phone Number and Credit Card Information.
- It is not possible to book a day that is in the past, those days are hidden or grayed out.
- Users are given error messages if the user input field is invalid
